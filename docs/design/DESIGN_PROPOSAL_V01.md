# Task Valley v0.1 디자인 브리프 및 구현 가이드

## 1. 레이아웃 방향

> **핵심 감각:** 화면을 열었을 때 흔한 투두(To-Do) 리스트가 아닌, '나의 어두운 도트 마을'이 가장 먼저 눈에 들어와야 합니다.

### 데스크톱 레이아웃 (1024px 이상)
- **전체 배경**: `#0f0f1a` (매우 어두운 밤하늘 색)
- **화면 분할**: 2단 컬럼 구조 (좌측 마을 중심, 우측 과제 패널).
  - **마을 영역 (좌측 60~70%)**: 화면의 감정적 중심입니다. 큰 2x2 그리드 형태의 마을 지도가 중앙에 넓게 위치합니다. 상단에는 상태 바(오늘 날짜, 전체 레벨, 연속 달성일)가 조용하고 절제된 타이포그래피로 자리 잡습니다.
  - **과제 패널 (우측 30~40%)**: 4개의 일일 과제 슬롯이 세로로 나열됩니다. 일반적인 모던 앱의 둥근 카드가 아닌, 도트 게임의 인벤토리나 상태창처럼 얇은 직선 테두리와 어두운 반투명 배경을 가집니다.

### 모바일 레이아웃 (390px 기준)
- **전체 배경**: `#0f0f1a`
- **화면 분할**: 상하 1열 구조.
  - **상단 (마을 영역)**: 상태 바 바로 아래에 마을 지도가 화면 너비를 꽉 채워 배치됩니다. 스크롤하기 전에 무조건 마을의 현재 상태(어두움 혹은 불빛)가 시야에 먼저 들어오도록 설계합니다.
  - **하단 (과제 패널)**: 마을 밑으로 스크롤하여 4개의 과제 슬롯을 확인합니다. 과제 완료 시, 상단으로 스크롤을 살짝 올렸을 때 마을에 불빛이 들어온 것을 바로 확인할 수 있는 높이감으로 배치합니다.

## 2. 컬러 팔레트 (Tailwind CSS 기준)

어두운 분위기에서 점진적으로 따뜻한 불빛이 켜지는 과정을 표현하기 위한 색상 토큰입니다. v0.1의 CSS 블록 구현을 위해 `tailwind.config.js`에 바로 반영할 수 있도록 구성했습니다.

```javascript
// tailwind.config.js 테마 확장
theme: {
  extend: {
    colors: {
      village: {
        bg: '#0f0f1a',          // 전체 화면 및 여백
        surface: '#1a1a2e',     // Stage 0 (버려진 상태)의 베이스 컬러 - 차가운 남색
        border: '#2a2a3d',      // 구역 간의 경계선 및 미완료 과제 테두리
        text: {
          primary: '#e8e8f0',   // 활성화된 텍스트
          muted: '#8888aa',     // 보조 텍스트 (단위, 미완료 목표 등)
        },
        energy: {
          amber: '#f59e0b',     // 불빛 (핵심 피드백 색상)
          stage1: '#2a2a3d',    // Stage 1 배경
          stage2: '#3d3020',    // Stage 2 배경 (어두운 황토색 계열로 웜톤 시작)
          stage3: '#5c4a1e',    // Stage 3 배경 (밝아진 황토색, 불이 켜진 느낌)
        }
      }
    }
  }
}
```

## 3. 마을 구역별 비주얼 메모

CSS `div`와 `grid`를 활용하여 2x2 형태의 마을을 구성합니다. 각 구역 안에는 창문이나 핵심 시설을 나타내는 작은 사각형(`w-2 h-2` 등) 블록들이 픽셀아트처럼 박혀 있습니다.

- **운동 (방어/훈련장)** [좌측 상단]
  - **형태**: 투박하고 각진 성벽이나 훈련 기둥의 실루엣. (CSS `border`를 두껍게 하거나 각진 계단형 블록 배치)
- **공부 (도서관/지식)** [우측 상단]
  - **형태**: 세로로 긴 직사각형이 꽂혀 있는 책장 실루엣. 
- **코딩 (작업장/공학)** [좌측 하단]
  - **형태**: 정사각형이 복잡하게 얽혀 있는 톱니바퀴나 서버랙 구조의 실루엣.
- **휴식 (공원/쉼터)** [우측 하단]
  - **형태**: 각진 형태 위주인 다른 구역과 달리 잎사귀를 표현하는 점진적인 블록 뭉치 형태. (유일하게 CSS 블록들에 여백을 더 줌)

> *v0.1 구현 팁*: 모든 구역의 색상을 다르게 하기보다, 구역의 형태적 차이만 CSS Grid로 주고 색상은 팔레트의 `energy` 톤으로 통일하는 것이 MVP 개발 속도를 높이고 감성을 통일하는 데 유리합니다.

## 4. Stage 0~3 상태 차이 (시각적 피드백)

과제 완료 버튼을 누를 때 해당 구역이 즉각적으로 변해야 합니다.

| Stage | 상태 | 배경색 | 시각적 특징 (CSS) |
|---|---|---|---|
| **0** | 버려진 상태 | `bg-village-surface` | 불이 모두 꺼진 검푸른 실루엣. 희미한 `border-village-border` 테두리. |
| **1** | 첫 불빛 | `bg-village-energy-stage1` | 구역 내 "창문" 역할을 하는 작은 `div` 블록 1개에 `bg-village-energy-amber` 적용. |
| **2** | 회복 중 | `bg-village-energy-stage2` | 웜톤으로 배경이 조금 밝아짐. 창문 `div` 2~3개에 불(`amber`)이 들어옴. |
| **3** | 활기 | `bg-village-energy-stage3` | 확연히 따뜻한 배경색. 여러 창문에 불이 들어오며 구역 전체에 약한 `box-shadow: 0 0 15px rgba(245,158,11,0.2)` 글로우(Glow) 효과 발생. |

## 5. Codex 구현 메모 (핵심 가이드)

1. **딜레이 없는 즉각적인 상태 전환 (가장 중요)**
   - 과제 리스트에서 완료 버튼 클릭 시, 마을의 구역 Stage 변경이 지연 없이 즉각적으로 화면에 반영되어야 합니다. API나 비동기 처리를 기다리지 않는 `localStorage` 기반이므로 무조건 **버튼을 누르는 타격감과 동시에 마을에 불이 켜져야 합니다.**
2. **트랜지션(Transition)으로 픽셀 불빛 감성 극대화**
   - 구역의 `bg-color`나 `box-shadow`가 변경될 때 `transition-colors duration-700 ease-in-out` 속성을 활용하세요. 딱딱하게 변하는 것보다 백열등이 스르륵 켜지는 듯한 부드러운 감성이 훨씬 매력적입니다.
3. **블록 조합 (Fake 픽셀아트 기법)**
   - 복잡한 이미지 에셋을 쓰지 않습니다. 큼직한 구역 컨테이너 안에 작은 `div`(예: `w-2 h-2`, `w-3 h-3`)들을 `absolute` 혹은 `grid`로 군데군데 배치하여 건물의 형태와 창문을 만드세요. Stage에 따라 이 작은 `div`들의 `bg`를 `bg-village-energy-amber`로 바꿔주면 훌륭한 픽셀 감성이 완성됩니다.
4. **과제 슬롯 (Task Item) 디자인 방향**
   - 모던한 모서리 둥글림(`rounded-lg` 이상)과 두꺼운 그림자를 피하세요.
   - `rounded-none` 또는 `rounded-sm` 수준의 얇은 테두리(`border border-village-border`)를 사용하세요.
   - **미완료 상태**: 투명한 바탕에 텍스트가 또렷하게 보임. 버튼은 눈에 띄게(`bg-village-border` 또는 하이라이트).
   - **완료 상태**: 과제 영역 전체의 투명도를 낮추고(`opacity-50`), 텍스트는 `text-village-text-muted`로 죽이며 취소선을 긋습니다. 버튼은 비활성화되고 테두리가 `border-village-energy-amber`로 바뀌어 완료를 축하하는 느낌을 줍니다.
