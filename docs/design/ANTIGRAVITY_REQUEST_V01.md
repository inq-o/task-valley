# Antigravity / Nano Banana 디자인 요청 v0.1

## 요청 목적

Task Valley MVP v0.1 구현 전에, Codex가 바로 참고할 수 있는 최소 디자인 산출물을 만들어 주세요.

이번 디자인은 완성형 게임 아트가 아니라 **Next.js + Tailwind + CSS 블록으로 구현 가능한 비주얼 방향**이어야 합니다.

## 먼저 읽을 문서

- `AGENTS.md`
- `docs/product/PRD.md`
- `docs/product/MVP_PLAN_V01.md`
- `docs/design/DESIGN_BRIEF.md`
- GitHub Issue #1

## 핵심 제품 감각

사용자가 과제 완료 버튼을 누르는 순간, 어두운 도트 마을의 특정 구역이 즉시 밝아져야 합니다.

첫 화면은 일반적인 할 일 앱이 아니라, **내가 회복시키는 작은 마을**처럼 보여야 합니다.

## 확정된 기능 범위

- 과제 슬롯은 4개 고정입니다: 운동, 공부, 코딩, 휴식.
- 과제 제목과 목표값은 v0.1에서 편집하지 않습니다.
- 마을 구역은 4개입니다:
  - 운동: 훈련장
  - 공부: 도서관
  - 코딩: 작업장
  - 휴식: 공원
- 각 구역은 4단계 성장 상태를 가집니다:
  - stage 0: 버려진 상태
  - stage 1: 첫 불빛
  - stage 2: 회복 중
  - stage 3: 활기 있는 상태
- 마을 전체 레벨은 4개 구역 stage 합계입니다.

## 요청 산출물

아래 항목을 한국어로 작성해 주세요.

1. **데스크톱 레이아웃**
   - 1024px 이상 화면 기준
   - 마을과 과제 목록의 배치
   - 상태 바 위치

2. **모바일 레이아웃**
   - 390px 폭 기준
   - 마을, 상태 바, 과제 목록의 세로 순서
   - 터치하기 쉬운 버튼 크기 제안

3. **컬러 팔레트**
   - 전체 배경
   - 패널 배경
   - 일반 텍스트
   - 보조 텍스트
   - 완료 강조색
   - stage 0~3 구역 색

4. **마을 구역별 비주얼 메모**
   - 훈련장, 도서관, 작업장, 공원이 각각 어떤 작은 픽셀 요소로 구분되는지
   - 예: 훈련장 = 깃발/목재 더미, 도서관 = 창문/책장, 작업장 = 굴뚝/기어 느낌, 공원 = 나무/벤치

5. **stage 0~3 상태 차이**
   - 각 단계에서 무엇이 추가되거나 밝아지는지
   - CSS 블록으로 구현 가능한 수준으로 제안

6. **Codex 구현 메모**
   - Tailwind class 방향
   - 레이아웃 우선순위
   - 구현하면 좋은 최소 애니메이션
   - v0.1에서 하지 말아야 할 디자인 요소

## 피해야 할 것

- 마케팅 랜딩 페이지
- SaaS 대시보드처럼 보이는 화면
- 큰 이미지 에셋 없이는 구현 불가능한 디자인
- 복잡한 캐릭터, NPC, 인벤토리, 상점, 맵 탐험
- 긴 튜토리얼 또는 별도 온보딩 화면
- v0.1 범위를 벗어나는 기능 제안
